# Hexxagon Desktop System

Полнофункциональная десктопная реализация классической настольной стратегической игры **Hexxagon** (популярная абстрактная игра на гексагональной дошці, схожая по механике с Ataxx). Проект написан на языке Python с использованием библиотеки **PyQt6** для построения графического интерфейса пользователя (GUI) и анимации игрового поля.

## О игре Hexxagon

**Hexxagon** — это пошаговая стратегическая игра для двух игроков, которая разворачивается на поле из шестиугольных ячеек (гексов). Цель игры — захватить как можно больше фишек на поле или полностью уничтожить фишки соперника до того момента, как дошка заполнится или ни один из игроков не сможет сделать ход.

### Основные правила и механика ходов:
1. **Клонирование (Хід на 1 ячейку):** Если вы перемещаете свою фишку на соседнюю пустую ячейку, фишка клонируется. Исходная фишка остается на месте, а в целевой ячейке появляется новая.
2. **Прыжок (Хід на 2 ячейки):** Если вы перемещаете фишку через одну ячейку (на расстояние двух гексов), фишка «прыгает» в целевую ячейку, а исходная ячейка становится пустой.
3. **Захват фишек:** После завершения любого хода (клонирования или прыжка) все смежные фишки соперника, находящиеся вокруг новой позиции (на расстоянии 1 гекса), перекрашиваются в цвет вашей команды (захватываются).
4. **Конец игры:** Матч завершается, когда:
   - Все поле полностью заполнено фишками.
   - Один из игроков потерял все свои фишки.
   - Ни один из игроков больше не имеет доступных ходов (в этом случае оставшиеся пустые места на поле автоматически засчитываются игроку с наибольшим числом фишек).

## Основной функционал

- **Два режима игры:**
  - `PvP (Против игрока)` — локальный мультиплеер за одним компьютером (Hotseat).
  - `PvE (Против бота)` — одиночная игра против встроенного алгоритма.
- **Встроенный игровой бот:** Поддерживает три уровня сложности, меняющих стратегию оценки ходов:
  - `Легкий` — делает случайный выбор из всех доступных ходов.
  - `Средний` — выбирает оптимальный ход по базовому алгоритму максимизации захвата.
  - `Сложный` — применяет строгую сортировку и приоритезацию ходов для максимального подавления соперника.
- **Интерактивное графическое поле:** Реализовано на базе `QGraphicsScene`. При клике на фишку подсвечиваются доступные для нее ходы:
  - `Зеленый цвет` — зона клонирования (расстояние 1).
  - `Фиолетовый цвет` — зона прыжка (расстояние 2).
- **Система сохранений:** Возможность в любой момент сохранить текущую сессию в файл `savegame.json` и продолжить игру после перезапуска приложения из главного меню.
- **Кастомный GUI:** Полностью бесфреймовые стилизованные диалоговые окна завершения игры со счетом матча и выбора уровня сложности.

## Архитектура и структура файлов

Проект логически разделен на три основных модуля:

* **`game_logic.py`** (`HexGameEngine`): Ядро игры. Отвечает за генерацию гексагональной сетки в аксиальных координатах `(q, r)`, расчет расстояний, правила перемещения фишек, захват соседей, подсчет очков и сериализацию/десериализацию данных в JSON.
* **`ai_bot.py`** (`HexAIBot`): Логика работы компьютерного оппонента. Рассчитывает веса возможных ходов на основе расстояния и количества потенциально захваченных фишек игрока.
* **`main.py`**: Главный модуль приложения. Содержит графические компоненты PyQt6 (`MainApplicationWindow`, `HexGridGameWidget`, `MainMenuWidget`), обрабатывает события ввода мыши и отрисовывает гексагональную сетку.

## Требования и установка

Для работы приложения необходим установленный интерпретатор **Python 3.8+** и библиотека **PyQt6**.
